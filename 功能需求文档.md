# Sized - macOS 窗口尺寸快速调整工具

## 功能需求文档 v2.0

---

## 1. 项目概述

### 1.1 应用名称
**Sized**

### 1.2 应用定位
一款 macOS 原生窗口管理工具，通过快捷键 + 轮盘交互方式，让用户能够快速将当前焦点窗口调整为预设的尺寸。采用 Swift + AppKit 原生开发，深度适配 macOS 26 Tahoe 的 Liquid Glass 设计语言，提供极致的原生体验。

### 1.3 目标平台
- **macOS 26.0 (Tahoe)** 或更高版本（主要适配 Liquid Glass 设计语言）
- **macOS 14.0 (Sonoma)** 起兼容（使用 fallback 样式）

### 1.4 目标用户
- 需要频繁调整窗口大小的开发者、设计师
- 追求高效窗口管理的 power user
- 喜欢使用快捷键和手势操作的用户
- 追求 macOS 原生体验的用户

### 1.5 核心价值
- **原生体验**：基于 AppKit 原生开发，与系统深度集成，性能卓越
- **极速操作**：一键呼出轮盘，鼠标滑动选择，无需记忆多个快捷键
- **高度可定制**：支持自定义轮盘样式和 8 个方向的窗口尺寸配置
- **Liquid Glass 适配**：率先适配 macOS 26 Tahoe 的全新设计语言

## 2. 核心功能

### 2.1 轮盘交互系统

#### 功能描述
用户按下全局快捷键后，在鼠标当前位置原地显示一个圆形轮盘界面，用户通过向对应方向移动鼠标来选择选项，释放鼠标或点击后执行对应的窗口尺寸调整操作。

#### 交互流程
```
1. 用户按下触发键（默认：右⌥ 或自定义修饰键组合）
2. 在鼠标光标位置显示轮盘 UI（NSPanel 浮动窗口）
3. 轮盘分为 8 个扇形区域（上、下、左、右 + 四个角）+ 中心区域
4. 用户移动鼠标到目标区域（距离 > 50pt 时选择方向，< 50pt 时选择中心）
5. 区域高亮显示，预览目标尺寸
6. 用户松开触发键确认选择
7. 当前焦点窗口调整为选定尺寸
8. 轮盘消失，操作完成
```

#### 触发机制（参考 Loop 实现）

**触发键系统**
- 支持单个或多个修饰键作为触发键（如右⌥、右⌃、⇪ 等）
- 支持侧边感知：区分左/右修饰键（如左⌥ 和右⌥ 可设为不同触发键）
- 触发延迟：可配置 0-500ms 的触发延迟，防止误触
- 双击触发：可选的双击触发模式（双击触发键激活轮盘）
- 超时自动关闭：触发键长时间未释放时自动关闭轮盘

**键盘快捷键触发**
- 使用 `CGEvent` tap 监听全局键盘事件（`keyDown`、`keyUp`、`flagsChanged`）
- 触发键 + 动作键组合：按下触发键后，再按方向键或其他键直接执行预设操作
- 支持绕过键绑定（Bypass Keybind）：直接用快捷键执行操作，无需先呼出轮盘
- 系统快捷键冲突检测：自动识别并放行系统级快捷键（如截图、Spotlight）

**鼠标中键触发**
- 通过 `CGEvent` 监听 `otherMouseDown` / `otherMouseUp` 事件
- 检测鼠标按钮编号为 2（中键）的事件
- 支持与键盘触发相同的延迟和双击配置

**鼠标交互（轮盘内的方向选择）**
- 使用 `PassiveEventMonitor` 监听 `mouseMoved` 和 `otherMouseDragged` 事件
- 计算鼠标相对于初始位置的角度和距离
- 距离阈值：
  - `directionalActionDistance = 50pt`：超过此距离选择方向区域
  - `noActionDistance = 10pt`：小于此距离视为无选择
  - 实际阈值 = `directionalActionDistance - radialMenuThickness`（轮盘厚度影响）
- 角度计算：将 360° 均分为 N 个方向区域，每个区域占 `360/N` 度
- 屏幕边缘补偿：当鼠标靠近屏幕边缘时，使用 delta 值继续跟踪移动

**左键点击**
- 在轮盘激活状态下，左键点击用于推进循环操作（Cycle）
- 通过 `ActiveEventMonitor` 监听 `leftMouseDown` 事件
- 仅响应来自 HID 系统状态的事件，避免误触发

#### 技术实现（参考 Loop 架构）

```swift
// 核心管理器：LoopManager（单例）
// 负责协调所有触发器和指示器
final class LoopManager {
    static let shared = LoopManager()
    
    // 触发器
    private(set) lazy var keybindTrigger: KeybindTrigger      // 键盘触发
    private(set) lazy var middleClickTrigger: MiddleClickTrigger  // 中键触发
    private(set) lazy var mouseInteractionObserver: MouseInteractionObserver  // 鼠标交互
    
    // 指示器服务
    private let indicatorService: WindowActionIndicatorService
    
    // 状态
    private(set) var isLoopActive: Bool = false
    private(set) var resizeContext: ResizeContext = .init()
}
```

### 2.2 轮盘窗口实现

#### 窗口类型
使用 `NSPanel`（`ActivePanel` 子类）作为轮盘的承载窗口：

```swift
@MainActor
final class ActivePanel: NSPanel {
    @objc dynamic var hasKeyAppearance: Bool { true }
    @objc dynamic var hasActiveAppearance: Bool { true }
}
```

#### 窗口配置
```swift
let panel = ActivePanel(
    contentRect: .zero,
    styleMask: [.borderless, .nonactivatingPanel],
    backing: .buffered,
    defer: true
)

// 关键属性
panel.ignoresMouseEvents = true          // 鼠标事件穿透
panel.collectionBehavior = .canJoinAllSpaces  // 所有桌面空间可见
panel.hasShadow = false                   // 无阴影（自定义阴影）
panel.backgroundColor = .clear            // 透明背景
panel.level = .screenSaver               // 最高层级，覆盖所有窗口
panel.contentView = NSHostingView(rootView: RadialMenuView(viewModel: viewModel))
```

#### 窗口定位
- **默认**：定位在鼠标光标位置（`NSEvent.mouseLocation`）
- **可选**：锁定在屏幕中心（`Defaults[.lockRadialMenuToCenter]`）
- 窗口大小 = `radialMenuSize + padding`（如 100 + 80 = 180pt）

#### 显示/隐藏动画
```swift
// 显示：scale(1.25) + opacity 过渡，0.1s
// 隐藏：opacity 过渡，0.15s，然后 orderOut + close
viewModel.setIsShown(true, animationDuration: 0.1)
viewModel.setIsShown(false, animationDuration: 0.15)
```

### 2.3 轮盘视图（RadialMenuView）

#### macOS 26 Tahoe 适配（Liquid Glass）

在 macOS 26 上，轮盘使用 Liquid Glass 材质渲染：

```swift
@available(macOS 26.0, *)
private func postTahoeView() -> some View {
    ZStack {
        // 使用 glassEffect 实现 Liquid Glass 效果
        radialMenuFill()
            .mask(directionSelectorMask)
            .glassEffect(
                .regular.tint(accentColorController.color1.opacity(0.025)),
                in: .rect(cornerRadius: radialMenuCornerRadius)
            )
            .mask(radialMenuMask)
        
        // 模拟 Liquid Glass 内边框
        RoundedRectangle(cornerRadius: radialMenuCornerRadius)
            .inset(by: radialMenuThickness - 0.5)
            .strokeBorder(lineWidth: 0.5)
            .foregroundStyle(borderColor)
            .mask {
                LinearGradient(
                    colors: [.white, .clear, .white],
                    startPoint: .topLeading,
                    endPoint: .bottomTrailing
                )
            }
    }
    .transition(.scale(scale: 1.25).combined(with: .opacity))
    .shadow(color: .black.opacity(0.2), radius: 10)
}
```

#### macOS 14-25 兼容样式（Pre-Tahoe）

```swift
private func preTahoeView() -> some View {
    ZStack {
        // 使用 NSVisualEffectView 实现毛玻璃效果
        VisualEffectView(
            material: .hudWindow,
            blendingMode: .behindWindow,
            state: .active
        )
        radialMenuFill()
            .mask(directionSelectorMask)
        radialMenuBorder()
    }
    .mask(radialMenuMask)
    .shadow(radius: 10)
    .opacity(viewModel.isShown ? 1 : 0)
}
```

#### 轮盘形状
- **圆形模式**（`radialMenuCornerRadius >= radialMenuSize / 2 - 2`）：
  - 使用 `Circle().strokeBorder()` 绘制环形
  - 方向选择器使用 `DirectionSelectorCircleSegment`（扇形 Path）
- **圆角矩形模式**：
  - 使用 `RoundedRectangle().strokeBorder()` 绘制环形
  - 方向选择器使用 `DirectionSelectorSquareSegment`（trim 裁剪）

#### 方向选择器
```swift
// 圆形扇形段（45° 扇形，以当前角度为中心）
struct DirectionSelectorCircleSegment: Shape {
    var angle: Double = .zero
    let radialMenuSize: CGFloat
    
    func path(in _: CGRect) -> Path {
        var path = Path()
        path.move(to: CGPoint(x: radialMenuSize / 2, y: radialMenuSize / 2))
        path.addArc(
            center: CGPoint(x: radialMenuSize / 2, y: radialMenuSize / 2),
            radius: radialMenuSize,
            startAngle: .degrees(angle - 22.5),
            endAngle: .degrees(angle + 22.5),
            clockwise: false
        )
        return path
    }
}
```

#### 中心图标
- 显示当前选中操作的 SF Symbol 图标
- macOS 26 上使用 `.symbolEffect(.drawOn)` 和 `.contentTransition(.symbolEffect(.replace))` 动画
- 无窗口时显示警告图标（`exclamationmark.triangle`）

## 3. 视觉界面设计规范

### 3.1 设计语言：macOS 26 Liquid Glass

#### 核心原则
- **内容优先**：界面控件轻盈，让用户专注于内容
- **物理真实感**：模拟真实玻璃的光学特性（折射、反射、透镜效应）
- **环境自适应**：界面元素根据背景内容、光线环境动态调整
- **圆角同心**：控件自动与容器保持圆角同心

#### Liquid Glass 材质
- **Regular 变体**：具备完整自适应与光效，推荐大多数场景
- **Clear 变体**：透明度更高，适合媒体内容丰富的背景
- 深色模式下透明度自动降低到 35%
- 窗口失焦时 Liquid Glass 自动"退后"

#### 设计规范
- **禁止"玻璃叠玻璃"**：避免在 Liquid Glass 上再叠一层 Liquid Glass
- 上层元素使用透明或活力填充（vibrancy），而不是再次套用 Liquid Glass
- 通过内置 `tint` 而不是 `background(Color)` 来设置色彩

### 3.2 主窗口布局

#### 采用 macOS 原生分栏布局
使用 SwiftUI `NavigationSplitView` 或 AppKit `NSSplitViewController`：

**左侧边栏（Sidebar）**
- 采用 Liquid Glass 材质，浮动在内容之上
- 包含红绿灯按钮（关闭、最小化、全屏）
- 导航菜单项列表
- 宽度：约 220-240pt

**右侧内容区（Content Area）**
- 无顶部标题栏边框
- 圆角卡片式内容展示
- 充足的内边距和留白
- 滚动边缘效果自动处理内容与浮动控件的可读性

### 3.3 窗口特性
- **标题栏**：红绿灯嵌入侧边栏顶部左侧
- **无传统标题栏**：采用自定义标题栏实现统一外观
- **圆角窗口**：macOS 原生支持的大圆角风格
- **毛玻璃效果**：
  - macOS 26：Liquid Glass 材质（`.glassEffect()`）
  - macOS 14-25：`NSVisualEffectView`（`NSHostingView` 包装）
- **动态背景**：支持浅色/深色模式自动切换（跟随系统主题）

### 3.4 控件风格
- **按钮**：胶囊形状（macOS 26 默认），支持 `.glass` 和 `.glassProminent` 样式
- **滑块**：支持刻度（tick marks），`neutralValue` 中点值
- **菜单**：图标统一在左侧
- **字体**：SF Pro（系统默认），回退 Inter
- **图标**：SF Symbols（优先），Lucide Icons（备选）
- **间距**：8pt 网格系统
- **动效**：流畅的过渡动画（0.25s ease），使用 SwiftUI `.animation()`

### 4.1 第一页：轮盘样式（Wheel Style）

#### 页面标题
**轮盘样式**

用于自定义轮盘的视觉外观，让用户根据个人喜好调整轮盘的显示效果。

| 配置项 | 类型 | 默认值 | 说明 |

- 分析当前壁纸的主色调，自动应用到轮盘
- 使用 `WallpaperProcessor` 进行图像分析
- 支持节流刷新，避免频繁计算

#### 实时预览
- 页面中央显示轮盘实时预览（使用 `RadialMenuView` 的 settings preview 模式）
- 所有参数调整即时反映到预览中
- 支持模拟鼠标 hover 效果演示

### 4.2 第二页：轮盘分配（Wheel Assignment）

#### 页面标题
**轮盘分配**

#### 功能描述
为轮盘的 8 个方向位置 + 中心位置分配不同的窗口操作。

#### 9 个位置定义

```
        [上 Top]
[左上]      [右上]
 LeftTop    RightTop

[左 Left]  [中心]  [右 Right]
           Center

[左下]      [右下]
LeftBottom  RightBottom
        [下 Bottom]
```

#### 可分配的操作类型

**1. 窗口方向操作**
- 半屏：上/下/左/右
- 四分之一屏：左上/右上/左下/右下
- 三分之一屏：左/中/右（水平）、上/中/下（垂直）
- 三分之二屏：左 2/3、右 2/3、上 2/3、下 2/3

**2. 通用操作**
- 全屏、最大化、近似最大化
- 居中、macOS 原生居中
- 最小化、隐藏
- 初始尺寸（恢复调整前的大小）
- 撤销

**3. 屏幕切换**
- 下一个屏幕、上一个屏幕
- 左/右/上/下方向的屏幕

**4. 窗口操控**
- 放大、缩小
- 向各方向收缩/扩展
- 向各方向移动

**5. 自定义尺寸**
- 支持用户输入自定义宽度和高度
- 单位：像素 (px) 或百分比 (%)
- 支持坐标定位模式（绝对坐标或百分比坐标）

**6. 循环操作（Cycle）**
- 为一个位置配置多个操作，每次触发时循环切换
- 例如：第一次左半屏 → 第二次左三分之一 → 第三次左三分之二

**7. 标签名称**（可选）
- 为该位置设置显示名称
- 显示在轮盘区域的图标中（SF Symbol）

#### 交互设计
- **可视化编辑器**：页面中央显示一个模拟轮盘
- 点击轮盘的某个区域可以选中并编辑该位置的配置
- 右侧或下方显示详细的配置面板
- 支持拖拽交换两个位置的配置
- 批量重置为默认值按钮

#### 数据持久化
- 使用 `Defaults` 库（基于 `UserDefaults` + iCloud 同步）
- 配置自动同步到 iCloud
- 支持导入/导出配置文件
- 提供恢复默认配置功能

### 5.3 关键技术点

#### 5.3.1 事件监控系统

**主动事件监控（ActiveEventMonitor）**
- 用于需要拦截/消费的事件（键盘快捷键、左键点击）
- 回调返回 `.ignore`（拦截）或 `.forward`（放行）
- 使用 `CGEvent.tapCreate` 创建事件 tap

**被动事件监控（PassiveEventMonitor）**
- 用于仅观察的事件（鼠标移动、中键点击）
- 不拦截事件，仅读取信息
- 使用 `NSEvent.addGlobalMonitorForEvents`

#### 5.3.2 轮盘窗口
- `NSPanel` 子类（`ActivePanel`），无边框、透明、非激活
- 窗口层级：`.screenSaver`（最高）
- 鼠标事件穿透：`ignoresMouseEvents = true`
- 跨空间可见：`.canJoinAllSpaces`
- 使用 `NSHostingView` 承载 SwiftUI 视图

#### 5.3.3 窗口调整
- 使用 Accessibility API (AXUIElement) 获取和调整窗口
- 通过 `AXUIElementCopyAttributeValue` 获取窗口属性
- 通过 `AXUIElementSetAttributeValue` 设置窗口位置和大小
- 需要辅助功能权限（Accessibility Permission）

#### 5.3.4 触觉反馈
- 使用 `NSHapticFeedbackManager` 提供触觉反馈
- 在方向选择变化时触发
- 支持禁用触觉反馈

#### 5.3.5 Liquid Glass 适配
```swift
// macOS 26+ 使用 glassEffect
if #available(macOS 26.0, *) {
    content.glassEffect(.regular.tint(color), in: .rect(cornerRadius: radius))
} else {
    // macOS 14-25 使用 NSVisualEffectView
    VisualEffectView(material: .hudWindow, blendingMode: .behindWindow, state: .active)
}
```

### 5.4 权限要求

#### 必需权限
- **辅助功能权限**（Accessibility）：用于检测和调整其他应用的窗口
  - 在 `Info.plist` 中声明 `NSAccessibilityUsageDescription`
  - 首次启动时引导用户授权
  - 使用 `AXIsProcessTrusted()` 检查权限状态

#### 可选权限
- **通知权限**：用于显示操作结果通知
- **iCloud 权限**：用于配置同步

## 6. 用户体验要求

### 6.1 性能指标
- 快捷键响应延迟 < 16ms（单帧）
- 轮盘显示动画 < 100ms
- 窗口调整完成时间 < 100ms
- 内存占用 < 30MB

### 6.2 可访问性
- 支持 VoiceOver
- 支持键盘完整操作（不仅依赖鼠标）
- 高对比度模式适配（Liquid Glass 自动适配）
- Reduced Transparency 模式适配
- Reduced Motion 模式适配

### 6.3 国际化
- 首选语言：简体中文、English
- 使用 `String(localized:)` 和 `Localizable.xcstrings` 管理国际化
- 所有用户可见文本支持本地化

### 6.4 错误处理
- 未获得辅助功能权限时显示引导提示
- 快捷键冲突时警告
- 窗口调整失败时给出明确错误信息
- 配置文件损坏时自动恢复默认
